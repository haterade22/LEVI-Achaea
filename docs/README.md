## Contributing Documentation

If you'd like to contribute documentation:

1. Follow the structure outlined above
2. Use clear, concise language
3. Include examples where appropriate
4. Keep documentation up-to-date with code changes

See [CONTRIBUTING.md](../CONTRIBUTING.md) for more details.

## Documentation Style Guide

- Use Markdown format
- Include code examples in appropriate language blocks
- Add screenshots for UI-related documentation
- Link to related documentation where relevant
- Keep line length to ~80 characters for readability

## Questions?

If you have questions about using LEVI or need clarification on documentation, please:

- Check existing documentation first
- Provide specific examples in your questions
